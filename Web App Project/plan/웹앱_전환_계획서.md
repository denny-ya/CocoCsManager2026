# Coco CS Manager — 웹앱 전환 계획서

> **작성일**: 2026-03-11  
> **목표**: React Native(Expo) 기반 모바일 앱을 **Google Workspace(Apps Script) 기반 웹앱**으로 전환  
> **핵심 원칙**: 현재 디자인·기능을 최대한 유지하면서, 설치 없이 모바일 브라우저에서 동작하는 웹앱 구현

---

## 1. 현재 앱 구조 요약

### 1-1. 화면(Screen) 목록

| # | 화면명 | 주요 기능 | 원본 파일 |
|---|--------|----------|----------|
| 1 | **홈** | 메뉴 카드 리스트, 각 화면으로 이동 | `app/index.tsx` |

| 4 | **AS 실적** | 월별·일별 AS 실적 대시보드 | `app/statistics/as-performance.tsx` |
| 5 | **배차 목록** | 날짜별 배차 조회, 상태(완료/이동중/대기) 표시 | `app/dispatch.tsx` |
| 6 | **영업점 주소 검색** | 영업점 검색, 전화 연결 | `app/store-directory.tsx` |

### 1-2. 공통 컴포넌트

| 컴포넌트 | 역할 |
|---------|------|
| `Header` | 파란색 상단 바 (뒤로가기 + 제목 + 홈 버튼) |
| `MenuCard` | 좌측 컬러바 + 아이콘 + 텍스트 카드 |
| `EmptyState` | 데이터 없음 안내 UI |
| `IconSymbol` | SF Symbols / Material Icons 래퍼 |

### 1-3. 디자인 시스템 (Colors.ts)

- **Primary**: `#0051A2` (진한 파랑)
- **Secondary**: `#0073CF` (밝은 파랑)
- **배경**: `#FFFFFF` / 카드: `#F5F5F5`
- 상태 색상: 성공(`#4CAF50`), 경고(`#FF9800`), 오류(`#F44336`)

### 1-4. 백엔드 (API 연동)

현재 이미 **Google Apps Script(GAS)** 를 백엔드로 사용 중:

| API | 기능 | 엔드포인트 파라미터 |
|-----|------|------------------|
| `searchBS` | BS/리워크 검색 | `action=search` |
| `getMasterStats` | 마스터 통계 | `action=masterStats` |
| `saveMemo` | 메모 저장 | `action=saveMemo` |
| `markAsComplete` | 점검 완료 | `action=complete` |
| `getAsPerformance` | AS 일일실적 | `action=asPerformance` |

→ **백엔드(GAS)는 변경 불필요**, 프론트엔드만 웹앱으로 전환하면 됩니다.

## 2. 전환 방식: Google Apps Script 웹앱

### 2-1. 왜 Apps Script 웹앱인가?

| 장점 | 설명 |
|------|------|
| **설치 불필요** | 모바일 브라우저에서 바로 접근 (URL 공유만으로 배포) |
| **Google Workspace 통합** | Google Sheets 백엔드를 직접 호출, 별도 서버 불필요 |
| **무료 호스팅** | Apps Script 자체가 호스팅 역할 |
| **빠른 업데이트** | 배포 버전만 갱신하면 즉시 반영 |
| **기존 API 재사용** | 현재 GAS 백엔드 코드를 그대로 활용 가능 |

### 2-2. 기술 스택

| 계층 | 기술 | 비고 |
|------|------|------|
| **프론트엔드** | HTML + CSS + JavaScript (Vanilla) | GAS `HtmlService`로 서빙 |
| **백엔드** | Google Apps Script (`doGet`/`doPost`) | 기존 API 로직 재사용 |
| **데이터** | Google Sheets | 기존 스프레드시트 그대로 사용 |
| **아이콘** | Material Icons (Google Fonts CDN) | SF Symbols 대체 |
| **폰트** | Pretendard 또는 Noto Sans KR (CDN) | 프리미엄 한글 폰트 |

## 3. 순차적 전환 계획

### Phase 1: 프로젝트 기초 설정 (1일)

- [ ] `Web App Project` 폴더 구조 생성
  ```
  Web App Project/
  ├── plan/              ← 계획 문서 (본 파일)
  ├── src/
  │   ├── Code.gs        ← GAS 서버 사이드 (doGet, API 라우터)
  │   ├── index.html     ← 메인 HTML (SPA 셸)
  │   ├── css/
  │   │   └── styles.css  ← 전체 스타일시트
  │   ├── js/
  │   │   ├── app.js      ← 라우터, 앱 초기화
  │   │   ├── api.js      ← GAS API 호출 래퍼
  │   │   └── pages/      ← 각 화면 JS
  │   │       ├── home.js
  │   │       ├── bs-search.js
  │   │       ├── statistics.js
  │   │       ├── as-performance.js
  │   │       ├── dispatch.js
  │   │       ├── store-directory.js
  │   │       └── work-guide.js
  │   └── components/     ← 재사용 UI 컴포넌트
  │       ├── header.js
  │       ├── menu-card.js
  │       └── empty-state.js
  └── documents/          ← 개발 문서
  ```
- [ ] GAS 프로젝트 생성 (`clasp` 또는 Apps Script 에디터)
- [ ] `appsscript.json` 설정 (webapp, oauthScopes)

### Phase 3: SPA 라우터 및 공통 컴포넌트 (1~2일)

### Phase 5: API 연동 전환 (1~2일)

- [ ] **기존 `fetch` 방식 → `google.script.run` 방식으로 전환**
  - 웹앱 내부에서는 `google.script.run.함수명()` 으로 서버 함수를 직접 호출
  - CORS 문제 완전 해소
  - 콜백: `withSuccessHandler()` / `withFailureHandler()`
- [ ] 기존 GAS 백엔드 함수 포팅 (또는 직접 호출)
  - `searchBS()` → `google.script.run.searchBS(keyword, part)`
  - `getMasterStats()` → `google.script.run.getMasterStats(master, part)`
  - `saveMemo()` → `google.script.run.saveMemo(vin, memo)`
  - `markAsComplete()` → `google.script.run.markAsComplete(vin, memo, processTypes)`
  - `getAsPerformance()` → `google.script.run.getAsPerformance(month, date)`
- [ ] 로딩 상태 / 에러 처리 통일

## 4. 준비사항 체크리스트


### 4-2. 기존 자원 확인
- [ ] 기존 GAS 백엔드 프로젝트 URL 확인
- [ ] Google Sheets 스프레드시트 ID 확인
- [ ] 기존 API 동작 테스트 (ping)

### 4-3. 전환 시 고려사항

| 항목 | React Native (현재) | 웹앱 (전환 후) | 비고 |
|------|---------------------|---------------|------|
| **네비게이션** | `expo-router` (Stack) | 해시 라우터 (`#/page`) | `history.pushState` 또는 해시 |
| **아이콘** | SF Symbols | Material Icons | CDN 로드 |
| **API 호출** | `fetch` + CORS 이슈 | `google.script.run` 직접 호출 | ✅ CORS 완전 해소 |
| **스타일링** | React Native StyleSheet | Vanilla CSS | CSS 변수 활용 |
| **상태 관리** | React `useState` | 전역 변수 또는 간단한 상태 객체 | SPA 패턴 |
| **전화 연결** | `Linking.openURL('tel:')` | `<a href="tel:">` | 동일하게 동작 |
| **애니메이션** | `LayoutAnimation` | CSS `transition` / `@keyframes` | 웹 표준 |
| **Safe Area** | `react-native-safe-area-context` | CSS `env(safe-area-inset-*)` | PWA 대응 |

### 4-4. 위험 요소 및 대응

| 위험 | 영향 | 대응 방안 |
|------|------|----------|
| GAS 실행 할당량 초과 | API 호출 제한 | 캐싱 + 호출 빈도 제한 |
| 초기 로딩 느림 | UX 저하 | 스켈레톤 로딩 UI + CSS/JS 최적화 |
| 브라우저 호환성 | 일부 기능 미동작 | 최신 CSS/JS만 사용, 폴리필 최소화 |
| 오프라인 미지원 | 인터넷 없으면 사용 불가 | PWA Service Worker (선택적 캐싱) |
