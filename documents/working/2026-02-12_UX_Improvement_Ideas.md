# 🎨 UX/UI 개선을 위한 추천 컴포넌트 제안

현재 구현된 UI를 바탕으로, 사용 편의성과 앱의 완성도를 높일 수 있는 컴포넌트들을 제안합니다. 이미 설치된 `react-native-paper` 라이브러리를 최대한 활용하는 방향입니다.

---

## 1. ⚡ 로딩 경험 개선: **Skeleton Loader (스켈레톤 로딩)**

> **현재**: 데이터 로딩 시 `ActivityIndicator`(뱅글뱅글 도는 로딩)만 표시됨.
> **제안**: 실제 콘텐츠의 모양(회색 박스)이 먼저 보이다가 데이터가 채워지는 방식.

- **적용 화면**: [검색] BS 리워크, 배차 목록, 영업점 목록
- **효과**: 체감 로딩 시간이 훨씬 짧게 느껴지고, 화면이 덜 깜빡이는 느낌을 줌.
- **난이도**: ⭐⭐ (보통)

## 2. 🔔 부드러운 알림: **Snackbar / Toast (토스트 메시지)**

> **현재**: 경고나 알림이 `Alert.alert()` (시스템 팝업)로 뜸. 흐름을 끊음.
> **제안**: 화면 하단에 검은색 바 형태로 잠시 떴다가 사라지는 메시지.

- **적용 예시**: "검색 결과가 없습니다", "전화번호가 복사되었습니다", "데이터가 새로고침 되었습니다"
- **효과**: 사용자의 작업을 방해하지 않으면서 정보 전달 가능.
- **난이도**: ⭐ (쉬움) - `react-native-paper`의 `Snackbar` 사용.

## 3. 🏷️ 빠른 필터링: **Chip (필터 칩)**

> **현재**: 검색 결과를 필터링하려면 별도 메뉴가 없거나 다시 검색해야 함.
> **제안**: 검색 결과 상단에 알약 모양의 버튼 나열.

- **적용 화면**: **[검색] BS 리워크 화면**
  - `[전체]` `[진행중]` `[완료]` `[대기]`
  - 터치 한 번으로 리스트 필터링.
- **효과**: 복잡한 검색 조건 입력 없이 빠르게 원하는 데이터만 볼 수 있음.
- **난이도**: ⭐ (쉬움) - `react-native-paper`의 `Chip` 사용.

## 4. 👆 당겨서 새로고침: **Pull to Refresh**

> **현재**: 데이터를 다시 불러오려면 검색 버튼을 다시 눌러야 함.
> **제안**: 리스트를 아래로 당기면 최신 데이터로 새로고침.

- **적용 화면**: 배차 목록, BS 검색 결과
- **효과**: 모바일 사용자에게 가장 익숙한 직관적인 새로고침 방식.
- **난이도**: ⭐ (쉬움) - `FlatList`의 기본 기능 (`onRefresh`) 활용.

## 5. 🆙 맨 위로 가기: **FAB (플로팅 액션 버튼)**

> **현재**: 리스트가 길어지면 맨 위로 스크롤하기 힘듦.
> **제안**: 스크롤을 내리면 우측 하단에 화살표(⬆️) 버튼 등장.

- **적용 화면**: 영업점 주소 검색 (데이터가 많음)
- **효과**: 긴 리스트 탐색 시 피로도 감소.
- **난이도**: ⭐ (쉬움) - `react-native-paper`의 `FAB` 사용.

## 6. 📅 직관적인 날짜 선택: **Bottom Sheet Calendar**

> **현재**: 날짜 입력이 텍스트(YYYY-MM-DD)나 기본 피커로 되어 있어 불편할 수 있음.
> **제안**: 화면 하단에서 달력이 부드럽게 올라와서 날짜를 선택.

- **적용 화면**: **배차 목록 화면**
- **효과**: 한 손으로 쉽게 조작 가능하며 앱이 훨씬 현대적으로 보임.
- **난이도**: ⭐⭐⭐ (어려움) - 별도 라이브러리(`react-native-calendars` 등) 필요할 수 있음.

---

## 📋 추천 적용 우선순위 (Phase 4에 포함 추천)

1.  **Pull to Refresh** (필수/쉬움) - 배차 목록에 즉시 적용
2.  **Chips** (추천/쉬움) - BS 검색 결과 필터링에 적용
3.  **Snackbar** (추천/쉬움) - 검색 실패/완료 메시지 대체
4.  **Skeleton** (옵션/보통) - 여유 있으면 적용
